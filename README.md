# 公認会計士試験 学習管理システム

Claude Code × GitHub で運用する、論文式まで見据えた学習管理システム。
記録するのは「やったこと」ではなく「**できるかどうか**」——証拠ベースで判定し、可視化する。

---

## このシステムが解こうとしている問題

短答の ○× だけでは「本当に解ける」かは分からない。

- 計算が合っていても、言語化できなければ論文で落ちる（＝ **隠れ弱点**）
- 答えを間違えても、原因が「計算」なのか「読解」なのか「理解」なのかで、直す場所がまったく違う

このシステムは、毎回の演習を **AI実力判定（A / B / C）** と **原因タグ** で記録し、復習を自動でスケジュールする。狙いは「正直な現在地」を常に見えるようにすること。

> **運用してわかった最大のボトルネックは、計算力ではなく読解（問題文・条件の読み取り）。**
> 計算は合うのに、条件を1行読み落として失点する——これを潰すのが最優先課題。

## 日々の使い方

1. **セッション開始** — Claude Codeに「今日の復習を生成して」と頼む → `progress/今日の復習.md` が更新される
2. **演習** — 今日のリストを上から解く（計算はタイマー必須）
3. **判定** — 解いた問題（手書き／音声書き起こしでも可）をAIに渡し、正直な A / B / C 判定を受ける
4. **記録** — 演習ログ・`weak-points.md`・`読解パターン集.md` に **追記**（上書きしない。解き直しは新しい行を足す）
5. **復習更新** — 判定に応じて `復習サイクル表.md` の予定日が振り直される
6. **push**

## 判定システム

### 2軸で管理する

- **作業ステータス**：⬜ 未着手 ／ 📝 ノート済 ／ 🔁 復習済
- **AI実力判定**：A / B / C（証拠ベース。口＝言語化、記＝記述 で確認する）

### 計算問題の判定（4指標）

`〇×` ／ `解答時間` ／ `周回数` ／ `解法想起`

→ **A** ＝ 正答 ＋ 時間内 ＋ 解法を自力で想起できる ＋ 複数回で安定

### 隠れ弱点

短答は ○ だが、言語化や解法手順が不確かなもの。「合ってるのに危ない」を可視化して、潰すまで追跡する。

### 原因の4区分（間違えたとき）

| 区分 | 意味 | 直す場所 |
|---|---|---|
| **計算ミス** | 処理は分かるが計算を誤った | 検算・手順の固定 |
| **読解ミス** | 概念は分かるが条件を読み落とした | 読解訓練（`読解パターン集.md`） |
| **理解不足** | そもそも処理を知らない | 理論・テキストに戻る |
| **時間切れ** | 解けるが間に合わない | 速度訓練 |

同じ1問でも原因が混在することがある（例：ある肢は読解、別の肢は理解）。**原因ごとに直す場所が違う**ので、必ず区別して記録する。

## 復習サイクル

スペースド・リピティション。判定結果に応じて予定日を自動で再設定する。

- 🔴 今日〜2日
- 🟡 今週
- 🟢 2週間〜1ヶ月

`今日の復習.md` は **予定日 ≤ 今日**（期限切れも含む）を拾い、1日あたり約10件に制限。並び順は「期限切れ → 今日」「🔴 → B → A維持」。やり残しを静かに消さないため、超過分は翌日に繰り越す。

---

## 重要度・難易度と捨て論点

各論点に **重要度 A/B/C ・ 難易度 A/B/C** を付与する。

**捨てるのは「重要度C × 難易度C」だけ。** 論点は独立に問える単位までスライスして管理する。

## Claude Code での運用

`CLAUDE.md` に運用ルールを置いてあり、Claude Code がセッション開始時に自動で読む。

AIは「正直な判定者」として動く——答えを甘くしない。間違いは間違いと記録し、隠れ弱点は隠れ弱点として残す。**心地よさより、正しい現在地。** それがこのシステムの一番大事なルール。
